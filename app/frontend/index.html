API Reference

Explore the payment endpoints

Four routes covering the full payment lifecycle — from card ingestion to settlement status.

💳
POST
/api/payment/
Process payment
Submit a card PAN with amount, currency, and merchant ID. Returns a tokenised transaction ID.
📋
GET
/api/transactions/
List transactions
Retrieve paginated transaction history. Supports skip/limit query params up to 100 records.
🔍
GET
/api/transactions/{token}
Get transaction
Look up a specific transaction by its UUID token. Returns full record including status.
🔧
PATCH
/api/transactions/{token}/status
Update status
Patch the status of a queued transaction. Used by Lambda after downstream validation.

Infrastructure

Built for businesses of every scale

PayCore runs on a hybrid Proxmox and AWS serverless stack. On-prem handles ingestion and storage. AWS handles the asynchronous settlement pipeline.

Proxmox on-prem

FastAPI + PostgreSQL run in Docker on a Proxmox VM. WireGuard VPN provides the secure tunnel to AWS.

AWS Lambda + SQS

Queued transactions are dispatched to SQS. Lambda consumers validate and settle, then PATCH status back via the API.

HashiCorp Vault

All secrets are dynamically issued by Vault. No static credentials in environment files or Terraform state.

PCI DSS alignment

PAN data is tokenised at ingestion. Raw card numbers never touch the database layer or log streams.
